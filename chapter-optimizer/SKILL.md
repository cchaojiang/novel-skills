# Chapter Optimizer Skill

章节优化师，负责在章节大纲基础上进行深度优化，丰富故事情节，增强阅读体验。

---

## 适用场景

- 对章节大纲进行二次优化
- 丰富情节细节，增强爽点
- 优化人物心理描写
- 增强伏笔埋入和钩子效果
- 调整节奏，提升张力
- 确保章节间衔接自然

---

## 输入

### 必须加载的文件

| 文件 | 说明 | 来源 |
|------|------|------|
| `volume_N_chapters.md` | 章节大纲（chapter-planner输出） | chapter-planner |
| `world_setting.md` | 世界观设定 | world-builder |
| `character_setting.md` | 人物设定 | world-builder |
| `foreshadow_table.md` | 伏笔追踪表 | novel-planner |
| `volume_outline.md` | 卷大纲 | novel-planner |

- `target_volume`：要优化的卷号（如：第1卷）
- `optimization_level`：优化级别（基础/中级/高级）

## 优化能力清单

### 1. 情节丰富化

```
原始大纲：情节简略，仅有骨架
优化后：情节血肉丰满，有血有肉

丰富要素：
- 场景细节描写（建筑/服饰/气候/声音/气味）
- 人物心理活动（犹豫/挣扎/决心/恐惧/喜悦）
- 对话内容增加（增加人物性格体现）
- 战斗场景细化（招数/特效/心理博弈）
- 情感铺垫增强（仇恨/爱慕/敬畏）
```

### 2. 爽点增强

```
原始爽点：简单带过
优化后：详细渲染，情绪到位

爽点类型增强：
- 打脸装逼：详细描述众人震惊反应
- 越级战斗：强调对手的强大，主角的逆袭
- 秘境夺宝：宝物详情，争夺过程的激烈
- 突破升级：异象描写，众人惊叹
- 势力崛起：成员数量，势力影响力
```

### 3. 伏笔优化

```
原始伏笔：简单标记
优化后：自然埋入，前后呼应

伏笔技巧：
- 日常细节埋入（不经意的一句话、一个动作）
- 对话中的暗示（人物对话暗藏玄机）
- 环境描写暗示（天气、景物对应情绪）
- 重复出现的事物（多次出现，暗示重要）
- 人物反应异常（反常行为预示未来）
```

### 4. 钩子优化

```
原始钩子：简单一句悬念
优化后：钩子尖锐，吸引眼球

钩子技巧：
- 危机突发（生死攸关时刻断章）
- 秘密揭示（关键信息只说一半）
- 身份暴露（人物真实身份即将揭开）
- 重大发现（宝物、遗迹、传承）
- 感情转折（爱恨情仇关键节点）
```

### 5. 节奏优化

```
原始节奏：平铺直叙
优化后：一张一弛，跌宕起伏

节奏调整：
- 紧张与舒缓交替
- 高潮与过渡章搭配
- 情感爆发点集中
- 战斗与日常平衡
- 信息密度调整
```

### 6. 人物塑造优化

```
原始人物：扁平化
优化后：立体多面

人物丰富：
- 性格多面性展示
- 内心矛盾与挣扎
- 成长变化过程
- 与他人互动细节
- 标志性动作/口头禅
```

### 7. 世界观融入

```
原始设定：干巴巴的说明
优化后：自然融入故事

融入技巧：
- 通过角色对话展示设定
- 场景描写体现世界观
- 修炼过程展示体系
- 势力互动展示格局
- 宝物描述展示资源
```

### 8. 衔接优化

```
原始衔接：简单"承接上一章"
优化后：自然过渡，有机衔接

衔接技巧：
- 场景转换自然（时间/空间过渡）
- 情绪延续（上章情绪自然过渡）
- 人物状态连贯（上章状态延续）
- 剧情线交织（多条线自然交替）
- 呼应前文（与前文细节呼应）
```

## 优化级别

### 基础级
- 情节完整，不出错
- 衔接自然
- 伏笔标记清晰
- 适合快速产出

### 中级
- 爽点增强
- 钩子优化
- 节奏调整
- 适合质量要求

### 高级
- 深度情节丰富
- 多维度优化
- 精品质量
- 适合重要章节

## 工作流

### 阶段1：分析（读取所有输入）
```
Step 1: load_volume_chapters    读取卷章节大纲
Step 2: load_world_setting     加载世界观设定
Step 3: load_character        加载人物设定
Step 4: load_foreshadow       加载伏笔表
Step 5: analyze_structure      分析章节结构
```

### 阶段2：分两次优化

#### 第一次优化：前半卷
```
Step 6: split_chapters         拆分章节为前半卷和后半卷
Step 7: optimize_first_half    优化前半卷章节
        - PlotEnhancer丰富情节
        - ForeshadowCraftsman优化伏笔
        - TensionMaster优化节奏和钩子
Step 8: verify_first_half      审核前半卷
```

#### 第二次优化：后半卷
```
Step 9: optimize_second_half   优化后半卷章节
        - 保持与前半卷结尾的衔接
        - 延续前半卷的节奏
        - 确保伏笔/钩子连贯
Step 10: verify_second_half   审核后半卷
```

### 阶段3：合并与审核
```
Step 11: merge_chapters        合并前半卷+后半卷
        - 确保中间章节流畅衔接
        - 验证伏笔/钩子连贯性
        - 检查节奏一致性
Step 12: quality_review       QualityReviewer最终审核
Step 13: revision            返工（如需要）
```

### 阶段4：输出
```
Step 14: save_optimized       保存优化后的章节大纲
Step 15: generate_report     生成优化报告
Step 16: upload_cloud        上传到飞书云文档
```

## 分两次优化的关键要求

### 章节拆分规则
- 总章节数为偶数时：前半卷 = 总章数/2，后半卷 = 总章数/2
- 总章节数为奇数时：前半卷 = (总章数+1)/2，后半卷 = (总章数-1)/2
- 示例：66章 = 前33章 + 后33章；67章 = 前34章 + 后33章

### 衔接要求
1. **章节编号衔接**：前半卷最后一章章号 +1 = 后半卷第一章章号
2. **情节衔接**：后半卷第一章的"衔接"项必须明确承接前半卷最后一章
3. **节奏衔接**：后半卷开头不能是高潮突然接入，需要平滑过渡
4. **伏笔衔接**：前半卷埋入的伏笔应在后半卷有呼应
5. **钩子继承**：前半卷结尾的钩子可以在后半卷开头揭示或延续

### 合并检查清单
- [ ] 前半卷最后一章的"钩子"与后半卷第一章的"衔接"对应
- [ ] 章节编号连续，无遗漏
- [ ] 伏笔埋入和揭示时间合理
- [ ] 节奏整体一致
- [ ] 人物状态连贯

## 优化技巧库

### 场景描写公式

```
公式：时间+空间+感官+氛围

示例：
- 晨光微露（时间），太初学宫的资质殿前（空间），聚集了数十名少年（氛围）
- 夕阳西下（时间），小镇的街道上（空间），人群熙熙攘攘（氛围）
```

### 心理描写公式

```
公式：刺激+反应+内心挣扎+决定

示例：
- 听到测试石异响（刺激），顾寒舟心中一紧（反应），"难道我的资质真的很差？"（内心），但他很快冷静下来，无论结果如何，他都不会放弃（决定）
```

### 爽点渲染公式

```
公式：事件+众人反应+对比+主角内心

示例：
- 顾寒舟被云老收徒（事件），众人倒吸凉气，"云长老从不轻易收徒！""这少年究竟有何特殊？"(众人反应)，其他少年投来羡慕嫉妒的目光（对比），顾寒舟表面平静，心中却热血沸腾（主角内心）
```

### 伏笔埋入公式

```
公式：日常细节+暗示+后续呼应点

示例：
- 云老临走时"无意间"看了顾寒舟胸口一眼（日常细节），这个细微的动作暗示他知道什么（暗示），后续在第XX章揭示玉佩秘密（呼应点）
```

## 与其他Skill的协同

```
chapter-planner（生成大纲）
        ↓
chapter-optimizer（优化大纲）← world_setting + character_setting
        ↓
chapter-writer（撰写正文）
```

### 协同要点
1. chapter-planner 输出基础大纲
2. chapter-optimizer 在大纲基础上进行丰富
3. chapter-writer 基于优化后的大纲撰写正文
4. 三者形成：基础→优化→创作 的完整流程

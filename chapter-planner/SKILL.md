# Chapter Planner Skill

章节详细大纲生成，用于将卷大纲细化为每卷的章节详细大纲。

---

## 适用场景

- 根据卷设计生成指定卷的每章详细大纲
- 每次只生成一卷的章节（按用户输入）
- 每章10项信息：标题/字数/人物/地点/情节(起承转合)/目标/爽点/伏笔/衔接/下章钩子

---

## 输入

### 必须加载的文件

| 文件 | 说明 | 来源 |
|------|------|------|
| `volume_outline.md` | 卷大纲（10卷完整内容） | novel-planner 输出 |
| `foreshadow_table.md` | 伏笔追踪表 | novel-planner 输出 |
| `world_setting.md` | 完整世界观设定 | world-builder 输出 |
| `character_setting.md` | 人物设定 | world-builder 输出 |
| `realm_system.md` | 境界体系 | world-builder 输出 |
| `cultivation_system.md` | 修炼体系 | world-builder 输出 |
| `factions.md` | 势力格局 | world-builder 输出 |
| `dungeons.md` | 秘境遗迹 | world-builder 输出 |

### 用户指定
- `target_volume`：要生成章节的卷号（如：第1卷）

### 输入处理流程

```
1. 从 volume_outline.md 提取 target_volume 的内容
   → 卷名/章节范围/核心剧情/主要人物/爽点/伏笔/结局

2. 加载伏笔表 foreshadow_table.md
   → 本卷需要埋入的伏笔

3. 加载世界观设定
   → 境界体系/修炼体系/势力格局/地图/秘境

4. 加载人物设定
   → 主角/配角/反派详细信息
```

### 质量要求

**⚠️ 核心要求：每一章都必须高质量！禁止前详后简！**

#### 绝对禁止
- ❌ 禁止前10章详细，后面的章简略（如"第XX-XX章"合并）
- ❌ 禁止用"..."省略情节
- ❌ 禁止跳过任何章节
- ❌ 禁止每10章只写一个小节

#### 每一章都必须详细
- ✅ 每章必须有独立的章节标题
- ✅ 每章必须有完整的10项内容
- ✅ 每章的情节(起承转合)必须详细展开
- ✅ 不能有任何章节被简化或合并
- ✅ 67章=67个独立详细章节

#### 质量标准
- 每章10项必须完整
- 每章情节起承转合都要详细
- 每章都要有具体的人物对话/心理描写
- 每章都要有具体的场景描写
- 每10章1个小高潮
- 每10章≥2个爽点
- 配角每卷出场≥10次
- 伏笔埋入必须明确
- 章节之间衔接自然

## Agent（3个）

| Agent | 职责 |
|-------|------|
| **ChapterOutlinePlanner** | 生成章节大纲（前半部分/后半部分） |
| **OutlineReviewer** | 校验每章高质量要求 |
| ** ContinuityChecker** | 校验前后半部分衔接 |

---

## 工作流（分两批次生成）

### 批次1：前半部分章节（1-XX章）
```
Step 1: extract_volume_info    提取当前卷设定
Step 2: generate_first_half    生成前半部分章节
Step 3: check_first_half      OutlineReviewer校验前半部分（每章都要高质量）
Step 4: save_first_half       保存前半部分
```

### 批次2：后半部分章节（XX+1-67章）
```
Step 5: load_first_half       加载前半部分内容
Step 6: generate_second_half  生成后半部分章节（必须承接前半部分剧情）
Step 7: check_second_half     OutlineReviewer校验后半部分（每章都要高质量）
Step 8: continuity_check      ContinuityChecker校验前后衔接
Step 9: merge_and_save        合并保存完整章节
```

### 关键要求
- 批次1生成完校验通过后才能生成批次2
- 批次2必须基于批次1的剧情自然衔接
- 每批次内部每章都必须高质量
- 前后半部分剧情必须连贯

## 章节格式（标准模板）

**每一章都必须详细完整，以下10项缺一不可：**

```markdown
### 第X章 章节标题

- **字数**: 3000-3500（必须达到）
- **人物**: xxx（身份，与主角关系）
- **地点**: xxx - 环境描写
- **情节**:
  - 起：xxx（开篇，场景切入，必须具体）
  - 承：xxx（发展，情节推进，必须详细）
  - 转：xxx（高潮，核心冲突，必须精彩）
  - 合：xxx（结尾，悬念/收尾，必须自然）
- **目标**: xxx
- **爽点**: xxx（1-2个）
- **伏笔**: xxx（如有，关联伏笔ID）
- **衔接**: 承接上一章xxx
- **钩子**: xxx
```

## 每章校验清单（必须逐章检查）

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 字数 | 3000-3500 |
| 2 | 人物 | 明确身份和关系 |
| 3 | 地点 | 具体场景描写 |
| 4 | 情节起 | 开篇必须具体 |
| 5 | 情节承 | 发展必须详细 |
| 6 | 情节转 | 高潮必须精彩 |
| 7 | 情节合 | 结尾必须自然 |
| 8 | 目标 | 清晰 |
| 9 | 爽点 | 1-2个 |
| 10 | 伏笔 | 明确埋入 |
| 11 | 衔接 | 自然连贯 |
| 12 | 钩子 | 有悬念 |

## 章节结构（每卷50章）

```
第一阶段：觉醒篇（1-10章）
第二阶段：修炼篇（11-20章）
第三阶段：崛起篇（21-30章）
第四阶段：转折篇（31-40章）
第五阶段：离别篇（41-50章）
```

## 生成示例（第1卷）

```markdown
### 第1章 异象降生

- **字数**: 3000
- **人物**: 林焰（婴儿）、林啸天、烈焰谷长老团
- **地点**: 烈焰谷内谷·产房
- **情节**:
  - 起：林焰出生之夜，烈焰谷上方天空异变，千里火烧云
  - 承：天地灵气暴走，所有灵灯熄灭，预言师惊惧
  - 转：婴儿降生时胸口黑光一闪而过，预言"灾星降世"
  - 合：林啸天抱住婴儿，眼神复杂，誓言保护
- **目标**: 建立主角身世背景，埋下灾星伏笔
- **爽点**: 异象之震撼，父亲守护之决心
- **伏笔**: V001绝焰之体、V007母亲下落
- **衔接**: 直接承接
- **钩子**: 预言师说"此子若留，烈焰谷必亡"，谷主何去何从？
```

## 7项校验（批次内校验）

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 完整性 | 每章10项齐全 |
| 2 | 衔接 | 章节连贯自然 |
| 3 | 伏笔 | 埋入+呼应 |
| 4 | 节奏 | 每10章1小高潮 |
| 5 | 爽点 | 每10章≥2个爽点章 |
| 6 | 人物 | 配角10-15章，反派20-30章 |
| 7 | 境界 | 突破在30-35或45-50章 |

---

## 前后半部分衔接校验（批次2开始前）

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 剧情连贯 | 后半部分首章承接前半部分结尾 |
| 2 | 人物一致 | 人物状态与前半部分一致 |
| 3 | 伏笔延续 | 前半部分伏笔在后半部分延续 |
| 4 | 节奏延续 | 高潮点连续 |
| 5 | 目标一致 | 章节目标与卷目标一致 |

# World Builder Skill

构建玄幻修仙小说的宏观世界设定。

---

## 适用场景

- 创建小说的基础世界观
- 设计修炼体系
- 设计人物设定
- 设计势力格局
- 这些设定将被 novel-planner（卷大纲）使用

---

## 输入（用户指定）

### 启动时需要询问的基本信息

```
在开始构建世界观前，需要向用户收集以下基本信息：

1. 小说标题（novel_title）
   - 例如：《逆天之路》《仙道永存》

2. 题材类型（genre）
   - 例如：玄幻修仙、仙侠、都市修仙

3. 主角设定（main_character）

   - 主角身份背景：
   - 主角性格特点：

4. 修炼体系（cultivation_system）
   - 修炼体系名称：

   - 境界数量：

5. 小说规模（scale）
   - 计划章节数：XXX章
   - 计划卷数：X卷

6. 特色设定（可选）

```

## 质量优化步骤

### 唯一性检查（必做）

在生成设定时，必须确保：

1. **姓名不重复**
   - 主角姓名避免常见：叶凡、林动、萧炎、韩立等
   - 配角姓名要有辨识度
   - 反派姓名要有威慑力

2. **势力名称独特**
   - 避免：星河宫、青云宗、天魔宗等常见名
   - 使用：太初学宫、幽冥渊、万宝阁等独特名

3. **功法体质创新**
   - 避免：星河诀、青云诀等常见名
   - 使用：太初道经、混元无极功等

4. **金手指创新**
   - 避免：古玉、戒指、珠子等常见金手指
   - 使用：昆仑玉玺、周天星辰图等

5. **避免套路化**
   - 不要"废物退婚"套路
   - 不要"戒指老爷爷"套路
   - 创新剧情线

## 工作流（10步）

### Step 0: 收集用户输入（必做）

```
0.1: 向用户询问基本信息
     - 小说标题
     - 题材类型
     - 主角设定（姓名/背景/性格）
     - 修炼体系（名称/属性/境界数）
     - 小说规模（章节数/卷数/是否需要仙界神界）
     - 可选（特殊体质/金手指/风格）

0.2: 汇总参数，向用户确认
     - 展示所有参数
     - 等待用户确认

0.3: 用户确认后，进入正式构建
     - 如果用户有修改，重新调整后再次确认
```

### Step 0.5: 唯一性优化

```
0.5: 唯一性检查与优化
     - 检查姓名是否与热门小说重复
     - 检查势力名称是否独特
     - 检查功法/体质/金手指是否创新
     - 如有重复，替换为独特名称
     - 确保错别字修正（如：瑕疵必报→睚眦必报）
```

### Step 0.6: 质量检查（新增）

```
0.6: 质量检查
     - 人物数量：665章至少需要8-10个主要人物
     - 势力数量：665章至少需要15-20个势力
     - 境界节奏：确保突破节点合理
     - 金手指功能：详细展开
     - 支线冲突：具体设计
     - 最终BOSS：必须包含在主要人物中
```

### Step 0.7: 标题生成（新增）

```
0.7: 标题生成
     - 生成1个主标题
     - 生成10个备用标题
     - 标题要体现小说核心特点
```

## 构建内容

### 1. 境界体系

```
9境+渡劫期（可根据需要调整）
- 淬体境 → 聚气境 → 化液境 → 金丹境 → 元婴境 → 化神境 → 炼虚境 → 合体境 → 大乘境 → 渡劫期

每境界需包含：
- 初期/中期/后期
- 实力定位
- 寿元
```

### 2. 修炼体系

```
灵气属性：
- 五行：金木水火土
- 变异：风、雷、冰、光、暗
- 特殊：空间、时间、灵魂

功法等级：
- 人阶 → 地阶 → 天阶 → 仙阶 → 神阶
- 每阶：黄/玄/地/天

特殊体质：
- 绝焰之体/九阴之体/先天道体
- 混沌体/轮回体/星辰体
- 血脉：神兽血脉/古仙血脉

灵根：
- 单灵根/双灵根/多灵根/变异灵根

其他体系：
- 炼丹/炼器/阵法/符箓
- 异火/异宝/灵兽/器灵
```

### 3. 势力格局

```
顶级势力（3-5个）
- 正道宗门
- 魔道宗门
- 中立势力
- 隐藏势力

中大型势力（5-10个）
- 家族（古族/新兴）
- 皇朝/帝国
- 商会/拍卖行
- 散修联盟
- 杀手组织

每势力需包含：
- 名称
- 所在地
- 领袖
- 核心功法
- 实力定位
```

### 4. 世界地图

```
界面划分：
- 下界（凡间/修真界）
- 仙界（可选）
- 神界（可选）

地域划分：
- 东域/南域/西域/北域/中州
- 海外/海域

货币体系：
- 下品灵石/中品灵石/上品灵石/极品灵石

重要设定：
- 秘境/遗迹（主角机遇来源）
- 天材地宝分布
- 危险区域（禁地/绝境）
```

### 5. 核心冲突

```
主线冲突：
- 修炼资源争夺
- 势力地盘划分
- 功法秘籍纷争
- 身世/仇恨

支线冲突：
- 门派争斗
- 家族恩怨
- 爱恨情仇
- 秘境探险竞争
```

### 6. 人物关系体系

```
主要人物（5-10个）
- 主角：身份/性格/目标/成长线
- 师父：身份/性格/与主角关系
- 朋友：身份/性格/与主角关系
- 爱人：身份/性格/与主角关系
- 敌人：身份/性格/与主角冲突
- 仇人：身份/性格/与主角血仇
- 贵人：身份/性格/帮助主角

人物关系图
- 主角 → 师父（师徒）
- 主角 → 朋友（兄弟）
- 主角 → 爱人（恋人）
- 主角 → 敌人（对抗）
- 师父 → 仇人（往事）
- 势力A → 势力B（敌对/联盟）

关系变化设计
- 前期关系
- 中期变化
- 后期反转
```

### 7. 800章扩展要素（可选）

```
根据小说规模可添加：
- 多个界面（下界/仙界/神界）
- 种族设定（人/妖/魔/鬼/仙）
- 气运/功德体系
- 天道/法则设定
- 远古秘宝/传承
- 轮回/转世设定
```

## Agent（8个）

| # | Agent | 职责 | 输出文件 |
|---|-------|------|---------|
| 1 | **RealmAgent** | 境界体系 | realm_system.md |
| 2 | **CultivationAgent** | 修炼体系 | cultivation_system.md |
| 3 | **FactionAgent** | 势力格局 | factions.md |
| 4 | **MapAgent** | 世界地图 | world_map.md |
| 5 | **ConflictAgent** | 核心冲突 | conflicts.md |
| 6 | **CharacterAgent** | 人物关系 | characters.md |
| 7 | **ExtensionAgent** | 扩展要素 | extensions.md |
| 8 | **WorldReviewer** | 校验所有设定 | review_report.md |

### 校验内容

**WorldReviewer 校验项：**
- 逻辑冲突检测
- 势力平衡性
- 境界合理性
- 人物关系一致性
- 世界观内部自洽

### 工作流步骤

```
Step 1:  RealmAgent       设计境界体系
         → 校验: WorldReviewer 检查逻辑

Step 2:  CultivationAgent 设计修炼体系
         → 校验: WorldReviewer 检查与境界匹配

Step 3:  FactionAgent     设计势力格局
         → 校验: WorldReviewer 检查势力平衡

Step 4:  MapAgent         设计世界地图
         → 校验: WorldReviewer 检查地域逻辑

Step 5:  ConflictAgent    设计核心冲突
         → 校验: WorldReviewer 检查冲突合理性

Step 6:  CharacterAgent   设计人物关系
         → 校验: WorldReviewer 检查人物关系一致性

Step 7:  ExtensionAgent   设计扩展要素（可选）
         → 校验: WorldReviewer 检查扩展合理性

Step 8:  merge_settings   合并所有文件 + 最终校验
         → 输出: world_setting.md + character_setting.md
         → 上传: 飞书云文档（使用write而非create+write两步）
```
